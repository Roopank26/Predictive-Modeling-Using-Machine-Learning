# Predictive Modeling Using Machine Learning

An end-to-end, reusable supervised machine learning project demonstrating the full
modeling workflow on a **Loan Approval** dataset (classification) with a secondary
**interest-rate** regression task.

> Raw Data → Data Cleaning → Feature Engineering → Encode/Scale → Train/Test Split →
> Model Training → Hyperparameter Tuning → Evaluation → Visualization → Prediction

---

## 📌 Project Overview

This repository is a portfolio-grade reference implementation of a complete supervised
ML pipeline. It covers every stage a production project needs: robust preprocessing
(no data leakage), feature engineering, comparison of multiple algorithms with
`GridSearchCV`, rigorous evaluation, automated visualization, and a reproducible
inference path. The code is modular (`src/`), documented, and runs identically on
Windows, Linux, and macOS.

The dataset is **synthetic** (generated by `src/generate_dataset.py`) so the project
is fully self-contained — no external downloads, accounts, or API keys required.

---

## ✨ Features

### Data Preprocessing (`src/preprocessing.py`)
- Drop duplicate records
- Impute missing values using **training-set statistics only** (median for numeric, mode for categorical) — avoids leakage
- Remove IQR outliers from the **training set only** (held-out test keeps its natural distribution)
- One-Hot encode categorical variables
- Scale features with `StandardScaler` (fitted on train only)
- Stratified train/validation/test split

### Feature Engineering (`src/feature_engineering.py`)
- New meaningful features: `debt_to_income`, `loan_to_income`, `income_per_dependent`, `loan_per_month`,
  `employment_ratio`, `credit_tier`, `age_group`
- One-Hot Encoding & Label Encoding demos
- Polynomial Features (degree 2)
- Feature selection: `SelectKBest` (univariate) and model-based selection

### Models (`src/train_model.py`)
**Classification:** Logistic Regression, Decision Tree, Random Forest, KNN, SVM, Naive Bayes
**Regression:** Linear Regression, Decision Tree, Random Forest

All trained with `GridSearchCV` (5-fold cross-validation, ROC-AUC / R² scoring),
parallelized with `n_jobs=-1`.

### Evaluation (`src/evaluate.py`)
- **Classification:** accuracy, precision, recall, F1, ROC-AUC + confusion matrix,
  ROC curve, precision-recall curve, feature importance, learning curve
- **Regression:** MAE, MSE, RMSE, R² + prediction-vs-actual plot, residual plot

## 💡 Usage

```bash
# 1. Generate the dataset (creates dataset/train.csv & test.csv)
python src/generate_dataset.py

# 2. Run the full pipeline (trains, tunes, evaluates, saves artifacts + reports)
python src/pipeline.py

# 3. Predict on a new record
python src/predict.py

# 4. Explore interactively
jupyter notebook notebooks/predictive_model.ipynb
```

### Programmatic prediction

```python
from predict import predict_new
import pandas as pd

sample = pd.DataFrame([{
    "age": 35, "income": 85000, "employment_years": 10, "loan_amount": 15000,
    "loan_term": 36, "credit_score": 720, "existing_loans": 1, "dependents": 2,
    "gender": "Male", "education": "Bachelor", "home_ownership": "Own",
    "loan_purpose": "Home", "marital_status": "Married",
}])
print(predict_new(sample))
#    approval_probability prediction
#                0.9027   Approved
```

The inference function re-applies the **exact** training pipeline
(impute with train stats → engineer → one-hot → scale) using the saved
`scaler.pkl`, `features.json`, and `imputation_stats.json`, so predictions are
consistent with training.

## 🔮 Future Improvements

- Swap the synthetic dataset for a real one (e.g. Titanic, Heart Disease, California
  Housing) by dropping `train.csv`/`test.csv` into `dataset/` and updating `src/config.py`.
- Add a `Pipeline`/`ColumnTransformer` to bundle preprocessing into a single, serializable
  scikit-learn object (simplifies deployment and eliminates manual schema alignment).
- Persist experiment metrics with MLflow or a CSV logger.
- Add unit tests (`pytest`) for each module and a CI workflow (GitHub Actions).
- Expose the model behind a small FastAPI service for live predictions.
- Tune the classification threshold on validation data to optimize for business cost
  (false-accept vs false-reject).

## 📝 Notes

- The dataset intentionally includes missing values, duplicates, outliers, and mixed
  feature types so every preprocessing step is exercised.
- Reproducibility: a fixed random seed is used throughout (`config.RANDOM_STATE`,
  dataset RNG, and all `train_test_split` calls).
- No secrets, credentials, or hardcoded absolute paths are present; all paths are
  derived relative to the project root.
